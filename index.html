១. ក្បួនសរសេរ Prompt

ដើម្បីឱ្យ Gemini ឆ្លើយតបបានល្អ និងចំគោលដៅ លោកគ្រូ-អ្នកគ្រូគួរប្រើរូបមន្ត Role + Task + Context + Format៖

💡 រូបមន្ត៖ តួនាទី (Role) + ភារកិច្ច (Task) + បរិបទ/លក្ខខណ្ឌ (Context) + ទម្រង់លទ្ធផល (Format)

  • Role (តួនាទី): ប្រាប់ឱ្យវាដើរតួជាអ្នកណា (ឧ. "អ្នកជាគ្រូបង្រៀនថ្នាក់ទី៧...")
  • Task (ភារកិច្ច): ប្រាប់ពីអ្វីដែលយើងចង់ឱ្យវាធ្វើ (ឧ. "ចូរបង្កើតលំហាត់ប្រាណ...")
  • Context (បរិបទ): ព័ត៌មានបន្ថែម ឬកម្រិតសិស្ស (ឧ. "សម្រាប់សិស្សទើបចាប់ផ្ដើមរៀនដំបូង...")
  • Format (ទម្រង់): ចង់បានលទ្ធផលជាអ្វី (ឧ. "ជាតារាង", "ជាចំណុចៗ", "ជាកិច្ចតែងការ")

ផ្នែកទី ៣៖ ឧទាហរណ៍អនុវត្តជាក់ស្តែងសម្រាប់គ្រូបង្រៀន (២៥ នាទី)

លោកគ្រូអាចយកល្បិច (Prompts) ទាំងនេះទៅបង្ហាញនៅលើអេក្រង់ផ្ទាល់ជូនលោកគ្រូ-អ្នកគ្រូ៖

គំរូទី ១៖ ការបង្កើតកិច្ចតែងការបង្រៀន (Lesson Plan)

"ចូរដើរតួជាគ្រូបង្រៀនផ្នែកបច្ចេកវិទ្យា (ICT)។ ចូរបង្កើតកិច្ចតែងការបង្រៀនមួយស្ដីពី 'មូលដ្ឋានគ្រឹះនៃការប្រើប្រាស់ Microsoft Word សម្រាប់សិស្សកម្រិតដំបូង'។ មេរៀននេះមានរយៈពេល ៤៥នាទី។ សូមរៀបចំឱ្យមាន៖ គោលបំណងមេរៀន, សកម្មភាពទី១ (ការណែនាំ), សកម្មភាពទី២ (ការអនុវត្តផ្ទាល់), និងសំណួរវាយតម្លៃនៅចុងម៉ោង។ សូមរៀបចំជាទម្រង់ចំណុចៗងាយអាន។"

គំរូទី ២៖ ការបង្កើតកម្រងសំណួរ ឬសន្លឹកកិច្ចការ (Quiz/Worksheet)

"ចូរបង្កើតសំណួរជ្រើសរើសចម្លើយត្រឹមត្រូវ (Multiple Choice) ចំនួន ៥ សំណួរ ស្ដីពី 'ប្រវត្តិប្រាសាទអង្គរវត្ត' សម្រាប់សិស្សថ្នាក់ទី៩។ សូមដាក់ចម្លើយជ្រើសរើស A, B, C, D និងផ្ដល់ចម្លើយត្រឹមត្រូវនៅផ្នែកខាងក្រោមបង្អស់។"

គំរូទី ៣៖ ការពន្យល់មេរៀនពិបាកៗឱ្យទៅជាងាយយល់

"ចូរពន្យល់ពី 'ដំណើរការរស្មីសំយោគរបស់រុក្ខជាតិ' ឱ្យងាយយល់បំផុត ដូចកំពុងនិយាយប្រាប់ក្មេងអាយុ ១០ឆ្នាំ។ សូមប្រើប្រាស់ការប្រៀបធៀបដែលងាយស្រួលស្រមៃ។"

ផ្នែកទី ៤៖ គន្លឹះកម្រិតខ្ពស់ និងការបង្កើត "Gems" ផ្ទាល់ខ្លួន (១០ នាទី)

  • ការបង្កើត Gems (Custom AI): ណែនាំលោកគ្រូ-អ្នកគ្រូពីរបៀបបង្កើត "Gems" ផ្ទាល់ខ្លួន (សម្រាប់គណនី Gemini Advanced) ដោយកំណត់មុខងារវាឱ្យទៅជា "ជំនួយការគ្រូបង្រៀនផ្ទាល់ខ្លួន" ដែលចាំតែជួយបង្កើតមេរៀន និងកែលំហាត់ឱ្យយើងជានិច្ច ដោយមិនបាច់វាយបញ្ជាដដែលៗឡើយ។
  • ការបញ្ចូលឯកសារ (Upload Files): លោកគ្រូ-អ្នកគ្រូអាចទាញទម្លាក់ (Drag & Drop) ឯកសារមេរៀនចាស់ៗ ឬរូបភាពចូលទៅក្នុង Gemini រួចបញ្ជាឱ្យវា៖
    • សង្ខេបអត្ថបទវែងៗ។
    • បង្កើតសំណួរចេញពីឯកសារនោះ។

២. ការងាររដ្ឋបាល & បង្រៀន

១. ផ្នែកការងាររដ្ឋបាល (Administration)

គោលបំណង៖ ជួយសម្រួលដល់ការសរសេរលិខិតផ្លូវការ ការរៀបចំរបៀបវារៈប្រជុំ ឬការសង្ខេបឯកសារ។

Prompt គំរូសរសេរលិខិតផ្លូវការ៖

"សូមដើរតួជាមន្ត្រីរដ្ឋបាលជំនាញម្នាក់។ ជួយសរសេរ លិខិតអញ្ជើញផ្លូវការ មួយច្បាប់ទៅកាន់ [ឈ្មោះបុគ្គល/ស្ថាប័ន] ដើម្បីចូលរួមក្នុងកម្មវិធី [ឈ្មោះកម្មវិធី] ដែលនឹងប្រព្រឹត្តទៅនៅថ្ងៃទី [កាលបរិច្ឆេទ] វេលាម៉ោង [ម៉ោង] នៅ [ទីកន្លែង]។ សូមប្រើប្រាស់ភាសាផ្លូវការ គួរសម ត្រឹមត្រូវតាមទម្រង់រដ្ឋបាលខ្មែរ និងមានរចនាសម្ព័ន្ធច្បាស់លាស់ (មាន យោង, គោលបំណង, និងខ្លឹមសារ)។"

Prompt គំរូរៀបចំរបៀបវារៈប្រជុំ (Meeting Agenda)៖

"សូមជួយរៀបចំ របៀបវារៈប្រជុំ (Meeting Agenda) សម្រាប់កិច្ចប្រជុំរយៈពេល ២ ម៉ោង ស្តីពី [ប្រធានបទប្រជុំ ឧទាហរណ៍៖ ការវាយតម្លៃលទ្ធផលការងារប្រចាំត្រីមាសទី១]។ សូមបែងចែកពេលវេលាឱ្យបានសមស្របសម្រាប់វគ្គនីមួយៗ រួមទាំងពេលពិភាក្សា និងសួរដេញដោល ដោយរៀបចំជាទម្រង់តារាងងាយស្រួលមើល។"

២. ផ្នែកការងារបង្រៀន (Teaching/Education)

គោលបំណង៖ ជួយគ្រូបង្រៀនក្នុងការរៀបចំកិច្ចតែងការបង្រៀន បង្កើតលំហាត់ ឬពន្យល់មេរៀនពិបាកៗឱ្យទៅជាងាយយល់។

Prompt គំរូបង្កើតកិច្ចតែងការបង្រៀន (Lesson Plan)៖

"សូមដើរតួជាគ្រូបង្រៀនដែលមានបទពិសោធន៍។ ជួយបង្កើត កិច្ចតែងការបង្រៀន (Lesson Plan) មួយសម្រាប់មុខវិជ្ជា [ឈ្មោះមុខវិជ្ជា ឧទាហរណ៍៖ អក្សរសាស្ត្រខ្មែរ] ថ្នាក់ទី [បញ្ជាក់ថ្នាក់ ឧទាហរណ៍៖ ទី៩] លើប្រធានបទ '[ឈ្មោះមេរៀន]' សម្រាប់រយៈពេល ៥០ នាទី។ នៅក្នុងកិច្ចតែងការ សូមរួមបញ្ចូល៖

• វត្ថុបំណងនៃការរៀនសូត្រ (Knowledge, Skills, Attitude)
• សកម្មភាពគ្រូ និងសកម្មភាពសិស្ស (ជំហានទី១ ដល់ ទី៥)
• វិធីសាស្ត្រវាយតម្លៃការយល់ដឹងរបស់សិស្សចុងម៉ោង។"

Prompt គំរូបង្កើតកម្រងសំណួរ/លំហាត់ (Quiz/Exercises)៖

"សូមបង្កើត សំណួរជ្រើសរើសចម្លើយត្រឹមត្រូវ (Multiple Choice Questions - MCQs) ចំនួន ៥ សំណួរ សម្រាប់សិស្សថ្នាក់ទី [បញ្ជាក់ថ្នាក់] លើមេរៀន '[ឈ្មោះមេរៀន]'។ សំណួរនីមួយៗត្រូវមានជម្រើស ៤ (A, B, C, D) និងសូមផ្តល់ចម្លើយត្រឹមត្រូវ រួមទាំងការបកស្រាយខ្លីៗនៅផ្នែកខាងក្រោមបង្អស់។"

៣. ការបង្កើត App Script

ជំហានទី ១៖ ចូលទៅកាន់ Script Editor

  • បើក Google Form របស់អ្នកដែលអ្នកចង់បន្ថែមផ្ទៃកូដ។
  • នៅជ្រុងខាងស្តាំផ្នែកខាងលើ សូមចុចលើ សញ្ញាចុចបី (More ⠇)។
  • អូសចុះក្រោម រួចជ្រើសរើសយកពាក្យ "Script editor" (កម្មវិធីកែសម្រួលស្គ្រីប)។ វានឹងបើកផ្ទាំងថ្មីមួយទៀតដែលជាផ្ទាំង Google Apps Script។

ជំហានទី ២៖ ដាក់បញ្ចូលកូដ (Paste Code)

  • នៅពេលដែលផ្ទាំង Apps Script លោតចេញមក អ្នកនឹងឃើញកូដដើមដែលគេដាក់ឱ្យស្រាប់ (ជាទូទៅគឺ function myFunction() { })។
  • សូម លុប កូដដើមនោះចោលទាំងអស់។
  • Paste (ដាក់បញ្ចូល) កូដ Apps Script ថ្មីរបស់អ្នកចូលទៅក្នុងផ្ទាំងនោះ។
  • នៅផ្នែកខាងលើខាងឆ្វេង អ្នកអាចចុចលើពាក្យ "Untitled project" ដើម្បីប្តូរឈ្មោះ Project របស់អ្នកឲ្យងាយស្រួលចំណាំ។
  • ចុចសញ្ញា Save (រក្សាទុក) ដែលជារូបថាសនៅរបារខាងលើ ឬចុច Ctrl + S (សម្រាប់ Windows) / Cmd + S (សម្រាប់ Mac)។

ជំហានទី ៣៖ ដំណើរការកូដ (Run) និងការអនុញ្ញាតសិទ្ធិ (Authorization)

ពេលខ្លះកូដរបស់អ្នកត្រូវការសិទ្ធិដើម្បីដំណើរការ (ឧទាហរណ៍៖ សិទ្ធិផ្ញើអ៊ីមែល ឬទាញយកទិន្នន័យ)។ ដើម្បីផ្តល់សិទ្ធិ៖

  • នៅរបារខាងលើ សូមចុចប៊ូតុង "Run" (ដំណើរការ)។
  • វានឹងលោតផ្ទាំងមួយផ្ទាំងបញ្ជាក់ពី Authorization required (ទាមទារការអនុញ្ញាតសិទ្ធិ) -> សូមចុច Review permissions។
  • ជ្រើសរើស គណនី Google (Gmail) របស់អ្នក។
  • ដោយសារវាមិនមែនជា App ផ្លូវការរបស់ Google វានឹងលោតផ្ទាំងព្រមាន។ សូមចុចលើពាក្យ Advanced (កម្រិតខ្ពស់) នៅផ្នែកខាងក្រោម។
  • បន្ទាប់មកចុចលើពាក្យ Go to [ឈ្មោះ Project របស់អ្នក] (unsafe)។
  • អូសចុះក្រោមបន្តិច រួចចុចពាក្យ Allow (អនុញ្ញាត) ជាការស្រេច។

ជំហានទី ៤៖ ការកំណត់ Triggers (ប្រសិនបើកូដទាមទារ)

ប្រសិនបើកូដរបស់អ្នករចនាឡើងដើម្បីឲ្យវាដំណើរការដោយស្វ័យប្រវត្តិ នៅពេលមានអ្នកចុច Submit Form (បញ្ជូនទម្រង់) អ្នកត្រូវកំណត់ Trigger៖

  • នៅរបារខាងឆ្វេងនៃផ្ទាំង Apps Script សូមចុចលើរូប នាឡិកា (Triggers)។
  • ចុចប៊ូតុង "+ Add Trigger" (បន្ថែមគន្លឹះ) នៅជ្រុងខាងស្តាំខាងក្រោម។
  • វានឹងលោតផ្ទាំងមួយមក សូមកំណត់ដូចខាងក្រោម៖
    • Choose which function to run: ជ្រើសរើសឈ្មោះ Function កូដរបស់អ្នក។
    • Select event source: ជ្រើសរើសយក "From form"។
    • Select event type: ជ្រើសរើសយក "On form submit" (មានន័យថាកូដនឹងរត់ពេលមានអ្នក Submit Form)។

៤. ការបង្កើត Web App

ដំណាក់កាលទី ១៖ ការរៀបចំរចនាសម្ព័ន្ធទូទៅ (Project Setup & Architecture)

ប្រើ Prompt នេះដើម្បីឲ្យ AI ជួយរៀបចំគម្រោង និងជ្រើសរើសបច្ចេកវិទ្យាដែលស័ក្តិសម។

"ខ្ញុំចង់បង្កើត Web App មួយសម្រាប់វិស័យអប់រំ ដែលមានមុខងារចម្បង៣គឺ៖
១. ធ្វើតេស្តអនឡាញ (Online Testing)
២. ហ្គេមក្នុងថ្នាក់ (In-class Games ដូចជា Kahoot) និង
៣. សកម្មភាពអន្តរកម្មប្លែកៗ (Interactive Activities)។

សូមជួយណែនាំ Tech Stack ដែលល្អបំផុតសម្រាប់បង្កើត Web App នេះ (ឧទាហរណ៍៖ React, Node.js, Firebase, Socket.io) ដោយផ្តោតលើភាពរលូនពេលសិស្សចូលលេងច្រើននាក់ (Real-time)។ បន្ទាប់មក សូមបង្កើតរចនាសម្ព័ន្ធថតឯកសារ (Folder Structure) សម្រាប់គម្រោងនេះ។"

ដំណាក់កាលទី ២៖ មុខងារធ្វើតេស្តអនឡាញ (Online Testing Module)

ប្រើ Prompt នេះដើម្បីបង្កើតប្រព័ន្ធប្រឡង ឬសួរដេញដោល។

"ផ្អែកលើគម្រោងខាងលើ សូមសរសេរកូដសម្រាប់ទំព័រ 'Online Test'។ មុខងារដែលខ្ញុំចង់បានរួមមាន៖

• មានសំនួរពហុជ្រើសរើស (Multiple Choice Questions) ដែលទាញយកពី Database។
• មាននាឡិកាកំណត់ម៉ោងរាប់ថយក្រោយ (Countdown Timer) សម្រាប់សំនួរនីមួយៗ។
• បង្ហាញពិន្ទុភ្លាមៗ (Auto-grading) នៅពេលសិស្សធ្វើតេស្តចប់។
• ចំណុចប្រទាក់ (UI) ត្រូវមានភាពទាក់ទាញ ងាយស្រួលមើល និងអាចប្រើបានយ៉ាងល្អនៅលើទូរស័ព្ទដៃ (Mobile Responsive)។"

ដំណាក់កាលទី ៣៖ មុខងារហ្គេមក្នុងថ្នាក់ (In-Class Games - Real-time)

នេះជាផ្នែកសំខាន់ដើម្បីបង្កើតភាពសប្បាយរីករាយ ដោយប្រើបច្ចេកវិទ្យា Real-time។

"ឥឡូវនេះ សូមសរសេរកូដសម្រាប់មុខងារ 'Live Class Game' ដែលដំណើរការជាលក្ខណៈ Real-time (សូមប្រើ Socket.io ឬ Firebase)។ លក្ខខណ្ឌមានដូចខាងក្រោម៖

• គ្រូជាអ្នកបង្កើតបន្ទប់ (Create Room) និងមានលេខកូដ (PIN Code) សម្រាប់ឲ្យសិស្សវាយបញ្ចូលដើម្បីចូលរួម។
• នៅពេលគ្រូចុច 'ចាប់ផ្តើម' សំនួរនឹងលោតនៅលើអេក្រង់របស់សិស្សទាំងអស់ក្នុងពេលតែមួយ។
• សិស្សដែលឆ្លើយត្រូវនិងលឿនជាងគេ នឹងទទួលបានពិន្ទុច្រើនជាង។
• មានតារាងពិន្ទុ (Leaderboard) បង្ហាញអ្នកលេខ១ លេខ២ និងលេខ៣ នៅចន្លោះសំនួរនីមួយៗ និងនៅពេលបញ្ចប់ហ្គេម។"

ដំណាក់កាលទី ៤៖ មុខងារសកម្មភាពប្លែកៗ (Unique Interactive Activities)

បន្ថែមមុខងារកម្សាន្ត និងបំបាត់ភាពធុញថប់ក្នុងថ្នាក់។

"ដើម្បីធ្វើឲ្យ Web App នេះកាន់តែប្លែក សូមជួយសរសេរកូដបង្កើត 'កញ្ចប់សកម្មភាពសប្បាយៗ' (Fun Activities Module) ចំនួន ២ មុខងារ៖

• កងបង្វិលរើសឈ្មោះ (Random Name Spinner): ជាកងរង្វង់ដែលគ្រូអាចបញ្ចូលឈ្មោះសិស្ស ហើយចុចបង្វិលដើម្បីហៅឈ្មោះសិស្សឲ្យឆ្លើយសំនួរដោយចៃដន្យ ជាមួយនឹង Effect សំឡេង និងរូបភាពអបអរសាទរ។
• ក្តារបញ្ចេញមតិរូបអារម្មណ៍ (Emoji Reaction Board): ជាទំព័រមួយដែលសិស្សអាចចុចបញ្ជូន Emojis (👍, ❤️, 👏, 😂) ពីទូរស័ព្ទរបស់ពួកគេ ហើយវានឹងហោះអណ្តែតពេញអេក្រង់ធំរបស់គ្រូក្នុងពេលបង្រៀន ដើម្បីបង្ហាញពីការចាប់អារម្មណ៍របស់ពួកគេ។"

៥. ការបង្កើត Gems

🎯 គោលបំណងមេរៀន៖

  • យល់ដឹងពីការកំណត់តួនាទី និងគោលដៅរបស់ Custom Gem (Google Gemini)។
  • ចេះប្រើប្រាស់ទម្រង់ Prompt ថ្នាក់ខ្ពស់ដើម្បីបង្កើត Instructions ដែលមានរចនាសម្ព័ន្ធច្បាស់លាស់។

១. សេចក្តីណែនាំ

ការបង្កើត Custom Gem ដែលឆ្លាតវៃ និងធ្វើការបានចំគោលដៅ គឺទាមទារឱ្យមានការផ្តល់ការណែនាំ (Instructions) ដ៏លម្អិត។ ខាងក្រោមនេះជាគំរូ Master Prompt ដែលអ្នកអាចយកទៅប្រើប្រាស់ដើម្បីឱ្យ AI ជួយរៀបចំគ្រោង Instructions ឱ្យបានល្អឥតខ្ចោះ។

២. គំរូ Prompt សម្រាប់យកទៅប្រើប្រាស់ (Copy ទៅប្រើក្នុង Gemini)

ចំណាំ៖ សូមប្តូរព័ត៌មាននៅក្នុងសញ្ញា [ ] ទៅតាមតម្រូវការជាក់ស្តែងរបស់អ្នក មុននឹងផ្ញើទៅកាន់ AI។

PROMPT TEMPLATE

I want to create a custom Gem. Please write a highly detailed, structured, and effective Instruction set for my Gem based on the details below.

Here are the details of my Gem:
- Name of the Gem: [ដាក់ឈ្មោះ Gem របស់អ្នកនៅទីនេះ]
- Role/Purpose: [ពន្យល់ពីតួនាទី ឬគោលបំណងរបស់វា ឧទាហរណ៍៖ ជាអ្នកជំនាញសរសេរ Content លក់អនឡាញ]
- Target Audience: [អ្នកប្រឡើប្រាស់ជាអ្នកណា? ឧទាហរណ៍៖ អាជីវករខ្នាតតូច, សិស្សនិស្សិត]
- Tone of Voice: [ទម្រង់សំឡេង/អារម្មណ៍នៃឆ្លើយតប ឧទាហរណ៍៖ បែបអាជីព (Professional), មិត្តភាព (Friendly), ឬ បែបកំប្លែង (Humorous)]
- Specific Rules/Constraints: [អ្វីដែលវាគួរធ្វើ និងមិនគួរធ្វើ ឧទាហរណ៍៖ ឆ្លើយតបជាភាសាខ្មែរជានិច្ច, មិនត្រូវប្រើពាក្យបច្ចេកទេសពេកទេ]

Please structure the Instructions into these sections:
1. Role and Objective: (Define who the Gem is and its ultimate goal)
2. Tone and Style: (Guidelines on how it should speak and format its answers)
3. Step-by-Step Process: (How it should process the user's request)
4. Constraints and Guardrails: (What it must avoid or never do)

៣. របៀបយកទៅអនុវត្តន៍

បន្ទាប់ពីអ្នកបញ្ជូន Prompt ខាងលើទៅកាន់ Gemini រួចរាល់ វានឹងបង្កើតទម្រង់ Instructions មួយយ៉ាងលម្អិត។ អ្នកគ្រាន់តែថតចម្លង (Copy) លទ្ធផលដែលទទួលបាននោះ យកទៅដាក់ក្នុងប្រអប់ Instructions ពេលបង្កើត Custom Gem ថ្មីរបស់អ្នកជាការស្រេច។

៦. បង្កើតក្នុង AI Studio

🎯 គោលបំណងនៃមេរៀន៖

  • យល់ដឹងពីអ្វីទៅជា Google AI Studio និងសារៈសំខាន់របស់វាក្នុងវិស័យអប់រំ។
  • ចេះប្រើប្រាស់ Prompt Interface ផ្សេងៗ (Chat, Freeform, Structured)។
  • អាចបង្កើត និងបណ្តុះបណ្តាល (Tune) ជំនួយការ AI ផ្ទាល់ខ្លួនសម្រាប់ជួយការងារបង្រៀន (ឧទាហរណ៍៖ AI ជួយបង្កើតកិច្ចតែងការ បង្កើតលំហាត់ ឬពន្យល់មេរៀន)។

ផ្នែកទី ១៖ សេចក្តីផ្តើម និងការស្គាល់ Google AI Studio

១. អ្វីទៅជា Google AI Studio?

Google AI Studio គឺជាឧបករណ៍ (Prototyping Environment) ផ្អែកលើ Web របស់ក្រុមហ៊ុន Google ដែលអនុញ្ញាតឱ្យយើងធ្វើការពិសោធន៍ និងបង្កើតកម្មវិធីបញ្ជា AI ដោយប្រើប្រាស់ម៉ូដែលដ៏ខ្លាំងពូកែគឺ Gemini Models។ វាផ្តល់ភាពងាយស្រួលក្នុងការបង្កើត Prompt ស្មុគស្មាញ និងការកំណត់លក្ខខណ្ឌឱ្យ AI ឆ្លើយតបតាមអ្វីដែលយើងចង់បាន។

២. ហេតុអ្វីលោកគ្រូ អ្នកគ្រូគួរប្រើប្រាស់ AI Studio?

  • ភាពបត់បែនខ្ពស់៖ អាចកំណត់ឱ្យ AI ដើរតួជាគ្រូបង្រៀនប្រចាំមុខវិជ្ជាណាមួយ ដែលមានសំដីទន់ភ្លន់ ឬម៉ឺងម៉ាត់បានតាមចិត្ត។
  • Context Window ធំទូលាយ៖ អាចដាក់ឯកសារមេរៀន (PDF, សៀវភៅទាំងមូល) ឬវីដេអូ ដើម្បីឱ្យ AI វិភាគ និងបង្កើតលំហាត់ចេញពីឯកសារនោះបានយ៉ាងត្រឹមត្រូវ។
  • ឥតគិតថ្លៃ (Free Tier)៖ មានកម្រិតប្រើប្រាស់ឥតគិតថ្លៃយ៉ាងទូលំទូលាយសម្រាប់សាកល្បង និងអភិវឌ្ឍ។

ផ្នែកទី ២៖ ការស្វែងយល់ពី Interface និងមុខងារសំខាន់ៗ

មុននឹងចាប់ផ្តើម សូមចូលទៅកាន់៖ aistudio.google.com ហើយកត់សម្គាល់ចំណុចសំខាន់ៗចំនួន ៣៖

១. ប្រភេទនៃ Prompt (Create New Prompt)

  • Chat Prompt: ប្រើសម្រាប់បង្កើតបទពិសោធន៍ជជែកពិភាក្សា (Conversational AI) ដូចជាការបង្កើត Chatbot សួរ-ឆ្លើយជាមួយសិស្ស។
  • Freeform Prompt: ផ្ទាំងទទេរដែលអនុញ្ញាតឱ្យយើងដាក់បញ្ចូលទាំងអត្ថបទ និងរូបភាព/វីដេអូ (Multimodal) ដោយសេរី សាកសមសម្រាប់បង្កើតមេរៀន ឬលំហាត់។
  • Structured Prompt: ការផ្តល់ឧទាហរណ៍ជាគំរូ (Few-shot prompting) ដើម្បីតម្រូវឱ្យ AI ឆ្លើយតបមកវិញចំទម្រង់ (Format) ដែលយើងចង់បានបេះបិទ។

២. ផ្ទាំងកំណត់ Config (ខាងស្តាំដៃ)

  • Model: ការជ្រើសរើសម៉ូដែល (ឧទាហរណ៍៖ Gemini 1.5 Pro សម្រាប់ការគិតស៊ីជម្រៅ ឬ Gemini 1.5 Flash សម្រាប់ល្បឿនលឿន)។
  • System Instructions: កន្លែងសំខាន់បំផុតសម្រាប់ដាក់ "ច្បាប់ ឬតួនាទី" ឱ្យ AI ដើរតួ (ឧទាហរណ៍៖ អ្នកគឺជាគ្រូបង្រៀនផ្នែកព័ត៌មានវិទ្យាថ្នាក់ទី១០...)។
  • Temperature: ការកំណត់ភាពច្នៃប្រឌិត (ជិត 0 = ចម្លើយត្រង់ៗតាមការពិត, ជិត 1 = ច្នៃប្រឌិតខ្ពស់)។

ផ្នែកទី ៣៖ ការអនុវត្តផ្ទាល់ (Hands-on Workshop)

🎯 គម្រោង៖ បង្កើត "ជំនួយការរៀបចំកិច្ចតែងការ និងលំហាត់ថ្នាក់ដំបូង"

ជំហានទី ១: ចុចលើ Create New Prompt រួចជ្រើសរើស Chat Prompt ឬ Freeform Prompt។

ជំហានទី ២: នៅប្រអប់ System Instructions (ខាងស្តាំ) សូមវាយបញ្ចូលអត្ថបទខាងក្រោម៖

"អ្នកគឺជាជំនួយការ AI សម្រាប់លោកគ្រូ អ្នកគ្រូថ្នាក់ទី៧ ដល់ទី១២។ ភារកិច្ចរបស់អ្នកគឺជួយបង្កើតផែនការបង្រៀន (Lesson Plan) សង្ខេបមេរៀន និងបង្កើតកម្រងសំណួរ/លំហាត់។ រាល់ចម្លើយរបស់អ្នកត្រូវតែ៖

- ឆ្លើយតបជាភាសាខ្មែរ ក្នុងទម្រង់ច្បាស់លាស់ ងាយស្រួលអាន (ប្រើប្រាស់ Bullet points)។
- ផ្តល់ឧទាហរណ៍ជាក់ស្តែងដែលទាក់ទងនឹងបរិបទរស់នៅប្រចាំថ្ងៃរបស់សិស្ស។
- បង្កើតសំណួរដែលមានកម្រិតគិតពីងាយទៅពិបាក។"

ជំហានទី ៣: ការសាកល្បងប្រើប្រាស់ (User Prompt) នៅក្នុងប្រអប់បញ្ចូលអត្ថបទ សូមសាកល្បងបញ្ជា៖

ឧទាហរណ៍ទី ១៖

"សូមជួយបង្កើតកិច្ចតែងការបង្រៀនខ្លីមួយស្តីពី 'ការប្រើប្រាស់ Microsoft Word សម្រាប់អ្នកចាប់ផ្តើមដំបូង' សម្រាប់រយៈពេល ៤៥ នាទី។"

ឧទាហរណ៍ទី ២ (Multimodal)៖ ផ្ទុកឡើង (Upload) រូបភាពដ្យាក្រាម ឬទំព័រសៀវភៅណាមួយ រួចបញ្ជាថា៖

"សូមពន្យល់ពីរូបភាពនេះ និងបង្កើតសំណួរពហុជ្រើសរើស (Quiz) ចំនួន ៣ ចេញពីវា។"

ផ្នែកទី ៤៖ បច្ចេកទេសកម្រិតខ្ពស់សម្រាប់គ្រូបង្រៀន

១. ការប្រើប្រាស់ Structured Prompt ដើម្បីកែវាយតម្លៃកិច្ចការសិស្ស

គ្រូបង្រៀនអាចប្រើ Structured Prompt ដើម្បីបង្រៀន AI ឱ្យស្គាល់ពីរបៀបដាក់ពិន្ទុ៖

Input (កិច្ចការសិស្ស)៖ [អត្ថបទដែលសិស្សសរសេរ]

Output (ការវាយតម្លៃ)៖ [ពិន្ទុ៖ .../១០ | ចំណុចល្អ៖ ... | ចំណុចត្រូវកែលម្អ៖ ...]

(នៅពេលយើងផ្តល់ឧទាហរណ៍គំរូ ២-៣ ទៅ AI វានឹងកែវាយតម្លៃកិច្ចការសិស្សបន្ទាប់ៗទៀតបានយ៉ាងត្រឹមត្រូវតាមស្តង់ដាររបស់យើង)

២. ការយកទៅប្រើប្រាស់ខាងក្រៅ (Get Code / Share)

នៅពេលលោកគ្រូអ្នកគ្រូបង្កើត Prompt ជោគជ័យហើយ អាចចុចប៊ូតុង "Get Code" ដើម្បីយកកូដនោះទៅដាក់ក្នុងកម្មវិធីផ្សេងៗ ឬប្រើមុខងារ Share ដើម្បីចែករំលែក Prompt នោះទៅកាន់មិត្តភក្តិរួមអាជីពប្រើប្រាស់បន្ត។

៧. ឧបករណ៍បង្កើត Prompt (គំរូ Gems)

ខាងក្រោមនេះជាគំរូសម្រាប់បង្កើត Custom Gems តាមមុខវិជ្ជា និងតម្រូវការជាក់ស្តែងរបស់អ្នកគ្រូ-លោកគ្រូ៖

ឧទាហរណ៍ទី ១៖ មុខវិជ្ជា «ភាសាអង់គ្លេស» (English Speaking Coach)

  • Name of the Gem: English Buddy
  • Role/Purpose: ជួយសិស្សហាត់និយាយភាសាអង់គ្លេសប្រចាំថ្ងៃ កែសម្រួលការបញ្ចេញសំឡេង (Pronunciation) និងកែទម្រង់វេយ្យាករណ៍ (Grammar) ឱ្យបានត្រឹមត្រូវ។
  • Target Audience: សិស្សខ្មែរដែលទើបចាប់ផ្តើមរៀននិយាយភាសាអង់គ្លេស។
  • Tone of Voice: បែបលើកទឹកចិត្ត (Encouraging) មិត្តភាព និងអត់ធ្មត់ខ្ពស់។
  • Specific Rules: ត្រូវឆ្លើយតបជាប្រយោគភាសាអង់គ្លេសខ្លីៗ និងងាយយល់ រួចបកប្រែជាភាសាខ្មែរនៅខាងក្រោម។ មិនត្រូវប្រើពាក្យពិបាកៗពេកទេ។

📝 គំរូ Prompt សម្រាប់កូពីទៅដាក់ក្នុង Gems System Instruction:

You are "English Buddy", an encouraging, friendly, and patient English speaking coach for Cambodian beginners. Your role is to help them practice daily English conversation, correct their pronunciation, and fix their grammar errors.

Strict Rules:
1. Always respond in short, simple, and easy-to-understand English sentences.
2. Below your English response, always provide a clear translation in Khmer.
3. Do not use complex vocabulary or advanced idioms. Keep it supportive and motivating.

ឧទាហរណ៍ទី ២៖ មុខវិជ្ជា «គណិតវិទ្យា» (Math Tutor)

  • Name of the Gem: លោកគ្រូ គណិត (Math Master)
  • Role/Purpose: ជួយពន្យល់លំហាត់គណិតវិទ្យា (រូបមន្ត មេគុណ អាល់ហ្សែប និងធរណីមាត្រ) ដល់សិស្ស។
  • Target Audience: សិស្សអនុវិទ្យាល័យ និងវិទ្យាល័យ។
  • Tone of Voice: បែបគ្រូបង្រៀន ច្បាស់លាស់ ម៉ឺងម៉ាត់ ប៉ុន្តែយល់ចិត្តសិស្ស។
  • Specific Rules: ហាមប្រាប់ចម្លើយភ្លាមៗ! ត្រូវពន្យល់ពីរូបមន្ត និងបង្ហាញវិធីធ្វើមួយដំណាក់កាលម្តងៗ (Step-by-step) ដើម្បីឱ្យសិស្សគិតតាម។

📝 គំរូ Prompt សម្រាប់កូពីទៅដាក់ក្នុង Gems System Instruction:

អ្នកគឺជា "លោកគ្រូ គណិត (Math Master)" ដែលជាគ្រូបង្រៀនគណិតវិទ្យាដ៏ច្បាស់លាស់ ម៉ឺងម៉ាត់ ប៉ុន្តែយល់ចិត្តសិស្ស។ ភារកិច្ចរបស់អ្នកគឺជួយពន្យល់លំហាត់គណិតវិទ្យា (រូបមន្ត មេគុណ អាល់ហ្សែប និងធរណីមាត្រ) ដល់សិស្សអនុវិទ្យាល័យ និងវិទ្យាល័យ។

ច្បាប់ដាច់ខាត៖
១. ហាមប្រាប់ចម្លើយចុងក្រោយទៅកាន់សិស្សភ្លាមៗជាដាច់ខាត!
២. ត្រូវពន្យល់ពីរូបមន្តដែលទាក់ទង និងបង្ហាញវិធីធ្វើមួយដំណាក់កាលម្តងៗ (Step-by-step) ដើម្បីណែនាំ និងអនុញ្ញាតឱ្យសិស្សគិតតាម និងដោះស្រាយដោយខ្លួនឯង។
